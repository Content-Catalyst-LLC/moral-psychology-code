# Validation Plan

This scaffold is synthetic and conceptual. Validation focuses on reproducibility, interpretive clarity, and responsible use.

## Checks

1. Scripts run from the article root folder.
2. Output folders are created automatically.
3. Generated tables are written to `outputs/tables`.
4. Figure outputs are written to `outputs/figures` where applicable.
5. Synthetic employees are not interpreted as real employees.
6. Unethical action probability is not interpreted as a real-person risk score.
7. Institutional integrity is not interpreted as a real audit or certification.
8. Accountability, voice, culture, and value alignment are documented as structural variables.
9. Code comments and docs clearly identify the data as synthetic.
10. Outputs are not used for screening, surveillance, discipline, hiring, promotion, or workplace decision-making.

## Possible empirical extensions

Future empirical work could include:

- de-identified organizational climate surveys
- speak-up and reporting-channel data
- ethics hotline categories
- audit and incident reporting data
- promotion and incentive-structure datasets
- employee psychological-safety measures
- culture and leadership assessments
- organizational misconduct case studies
- qualitative interview coding schemes
- institutional purpose-drift indicators

Any empirical extension should include source documentation, privacy safeguards, ethical review where appropriate, measurement validation, and careful interpretation.
